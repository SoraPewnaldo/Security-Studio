# AES Encrypt/Decrypt

## Overview

A robust tool for encrypting and decrypting text using Advanced Encryption Standard (AES) algorithms, strictly relying on the browser's Web Crypto API.

## Features

- **AES-GCM**: Authenticated encryption (protects both confidentiality and integrity). This is the recommended mode.
- **AES-CBC**: Legacy block cipher mode.
- Key derivation using PBKDF2 with 100,000 iterations to protect against brute-force attacks.
- Everything runs client-side. No data is sent to a server.

## How it Works

1. **Key Derivation**: The tool takes your password and a randomly generated 16-byte salt, then runs it through PBKDF2 (SHA-256, 100,000 iterations) to derive a secure 256-bit AES key.
2. **Encryption**: It generates a random Initialization Vector (IV). The data is encrypted using the derived key and IV.
3. **Packaging**: The output Ciphertext is a Base64 encoded string containing the concatenation of: `[16-byte Salt] + [12 or 16-byte IV] + [Encrypted Payload]`.
4. **Decryption**: To decrypt, it reverses the process: extracting the Salt and IV from the beginning of the Base64 payload, deriving the key, and decrypting the remaining bytes.

## Examples

If you encrypt "Hello World" with the password "secret123", you will receive a Base64 string. Due to the random Salt and IV, encrypting the exact same text with the exact same password will yield a completely different Base64 string every time.

## Limitations

- The ciphertext format (`Salt + IV + Payload`) is specific to this tool's packaging logic, though it's a very standard approach. It may not directly decrypt payloads generated by other tools (like `openssl`) unless they use the exact same packaging and key derivation strategy.
