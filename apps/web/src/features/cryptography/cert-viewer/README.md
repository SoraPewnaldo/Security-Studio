# Certificate Viewer

## Overview

Decode and inspect X.509 certificates directly in your browser. This tool helps security engineers and developers understand the contents of SSL/TLS certificates without needing command-line tools like OpenSSL.

## Features

- Parses PEM-encoded X.509 certificates (`.crt`, `.pem`)
- Displays Subject and Issuer Distinguished Names (DN)
- Computes SHA-256 fingerprint (thumbprint) natively
- Extracts Subject Alternative Names (SANs) including DNS and IP
- Validates expiration dates and clearly highlights if the certificate is expired

## How it Works

The tool uses the `@peculiar/x509` library to parse the ASN.1 structure of the certificate. 
- Thumbprints are generated by hashing the raw DER bytes of the certificate using the Web Crypto API (`crypto.subtle.digest`).
- SANs are extracted by looking up the specific OID `2.5.29.17` (Subject Alternative Name) in the certificate's extensions array.

## Examples

Paste a standard PEM block:
```text
-----BEGIN CERTIFICATE-----
MII...
-----END CERTIFICATE-----
```
The tool will immediately parse and display the human-readable fields.

## Limitations

- Currently only supports single certificates (does not parse full certificate chains / bundles at once).
- Does not cryptographically verify the certificate against a Root CA trust store (it only parses the data).
